## 4. 미충족 니즈 발굴

### 기존 솔루션의 공통 격차 (Gap Analysis)

1. **주보 자동 생성 불가**: 매주 수작업으로 제작. 행사 일정·설교 제목·헌금 명단을 데이터에서 자동 조합하는 솔루션 없음
2. **새신자 케어 자동화 부재**: 등록 후 후속 돌봄(방문 스케줄, 안내 문자, 소그룹 매칭)이 모두 수동
3. **보고서 수동 생성**: 당회/제직회/총회 보고서를 간사가 수집·취합·서식화에 수 시간 소요
4. **봉사자 배정 최적화 없음**: 은사·스케줄·이력 기반 최적 배정 로직 없음
5. **재정 anomaly 감지 없음**: 이상 헌금 패턴, 예산 초과 예고 등 지능형 분석 부재
6. **다중 시스템 사일로**: 카카오채널, 구글캘린더, 엑셀, 교적 시스템이 연동 안 됨
7. **자연어 데이터 조회 불가**: "지난달 결석 3회 이상 교인 목록" 같은 질문을 UI 클릭 없이 처리 불가

### 잠재 고객의 실제 니즈 vs 현재 제공

| 페르소나 | 실제 니즈 | 현재 제공 | 갭 |
|----------|-----------|-----------|-----|
| 담임 목사 | "AI가 행정 처리, 나는 말씀·성도 집중" | 데이터 조회 시스템 | 완전한 자동 실행 부재 |
| 행정 간사 | "주간 반복 업무(주보, 문자, 보고서) 자동화" | 데이터 입력 도구 | 작업 자동화 불가 |
| 장로·집사 | "재정 현황 실시간 파악, 이상 감지" | 수동 리포트 조회 | 지능형 분석 부재 |
| 셀/목장 리더 | "소그룹 출석·기도제목 자동 취합" | 앱 수동 입력 | 집계 자동화 없음 |

### AI Agentic Workflow가 제공할 수 있는 독특한 가치

1. **Zero-touch 자동화**: 주보 자동 생성, 새신자 케어 파이프라인, 당회 보고서 자동 초안
2. **대화형 행정**: "이번 달 헌금 감소 교인 목록 + 개인 문자 초안 작성해줘" → 한 문장으로 실행
3. **교차 시스템 오케스트레이션**: 카카오, 캘린더, 재정, 교적을 하나의 에이전트가 조율
4. **예측적 돌봄**: 출석 감소 패턴 감지 → 자동 심방 스케줄 제안

## 5. 최종 결론

### 시장 기회의 크기: **중간-큼** (Medium-Large)

**근거:**
- 타겟 가능 교회 15,000~20,000개 (중형 + 행정 의지 있는 소형)
- 목회자 79.9%가 AI 사용 의향 (수요 검증됨)
- 기존 시장 공급 공백: 경쟁사 5개 모두 AI 자동화 전무
- 한국 ChMS 시장 USD 14.6M 성장세 + AI 카테고리 프리미엄 가산 가능
- 단, 교회 예산 민감도가 높아 무료 또는 저가 진입 전략 필요

### 진입 시점: **지금이다 (NOW)**

**근거:**
- 2025→2026년이 "AI를 어떻게 쓸 것인가" 단계로 전환 중 (목회데이터연구소)
- 목회자 AI 사용률 2년 내 2배 증가 (40.7% → 79.9%) — 모멘텀 명확
- 경쟁사 중 AI 자동화 시도 업체 없음 — 레드오션이 아닌 신규 카테고리 창출 기회
- Claude Code (claude-sonnet-4-6)의 성숙도 + agentic workflow 기술 성숙 시점 일치
- 교단별 디지털화 압력 증가 (교인 감소 속 효율화 니즈)

### 우리의 차별화 3가지

1. **Zero-touch 행정 자동화**: 기존 솔루션이 "입력 도구"라면, 우리는 "실행 에이전트" — 주보 자동 생성, 새신자 자동 케어 파이프라인, 보고서 자동 초안 등 반복 업무를 에이전트가 직접 수행
2. **대화형 교회 행정 인터페이스**: 목사·간사가 자연어로 명령 → AI 에이전트가 교적+재정+캘린더+카카오를 오케스트레이션 — 기존 솔루션은 클릭·입력 중심
3. **Cross-system 통합 오케스트레이션**: 카카오채널, 구글캘린더, 재정 시스템, 교적 시스템을 MCP 서버로 통합한 단일 자동화 파이프라인 — 현재 시장에 존재하지 않는 기술 계층

### PRD에 반영해야 할 시장 신호

| 시장 신호 | PRD 반영 방향 |
|-----------|--------------|
| 목회자 AI 사용 의향 1위: 교회 행정 전산화 (63.9%) | 행정 자동화를 핵심 기능으로 최우선 설계 |
| 무료 경쟁사 다수 (치윰, 오직, 온맘) | Freemium 진입 전략: 소형 무료 → 중형 유료 자동화 기능 |
| 예장합동·통합 각 1만개 교회 | 교단 파트너십 채널이 빠른 확산 경로 |
| 2026 트렌드: "심플처치" (조직 단순화) | 관리 부담 축소 메시지로 포지셔닝 |
| 봉사자 체력/시간 부담 (30%, 22%) | 봉사자 배정 최적화 + 반복 업무 자동화 기능 |
| 카카오 채널이 교회 소통 표준 | 카카오 MCP 통합이 필수 기능 |
| 중형 교회: IT 중요성 인식하나 체계적 지원 부족 | 설치·운영 복잡도 최소화 (Complexity-free onboarding) |
