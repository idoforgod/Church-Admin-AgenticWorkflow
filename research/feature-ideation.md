## 1. 글로벌 ChMS 벤치마킹 요약

### 1.1 벤치마킹 대상 플랫폼

| 플랫폼 | 핵심 강점 | AI 현황 | 한계 |
|--------|----------|---------|------|
| **Planning Center** | 예배 계획, 봉사자 스케줄링, 체크인 | 부분적 (보고서 자동화) | AI 자동화 없음, 모듈 각각 유료 |
| **Breeze ChMS** | 사용 편의성, 교인 데이터베이스, 헌금 | 기초 자동화 | 보고서 생성 수동 |
| **ACS Technologies** | 포괄적 기능 (급여, 재정, 교인), 종합 보고서 | 없음 | 복잡성 높음, 소형 교회 부적합 |
| **Tithe.ly** | 헌금·기부 특화, 다채널 수납 (앱/문자/키오스크) | 기초 분석 | 행정 기능 약함 |
| **ChurchTrac** | 올인원 플랫폼, 저가 | 없음 | 깊이 부족 |

### 1.2 글로벌 ChMS의 공통 기능 목록

```
교인 데이터베이스 (People DB)
├── 가족 단위 정보 관리
├── 세례/학습/입교 기록
├── 봉사 이력 추적
├── 그룹/소그룹 배정
└── 방문자 추적

헌금 & 재정
├── 온라인/앱/키오스크 헌금 수납
├── 정기 헌금 자동화
├── 기부금 영수증 발행
├── 펀드별 지정 헌금 추적
└── 재정 보고서

예배 & 봉사자
├── 예배 순서 계획
├── 봉사자 스케줄링 (자동 알림)
├── 음악 악보 관리
└── 체크인 시스템

커뮤니케이션
├── 이메일/SMS 대량 발송
├── 그룹별 타겟 메시지
├── 자동화 후속 이메일
└── 모바일 앱 푸시 알림

보고서
├── 출석 추세 보고서
├── 헌금 현황 보고서
├── 그룹 참여도 보고서
└── 사용자 정의 보고서
```

### 1.3 AI Agentic Workflow로 더 잘할 수 있는 것

글로벌 ChMS는 **데이터 기록 + 수동 보고서 조회** 수준에 머물러 있다. AI 에이전트가 추가하는 계층은 다음과 같다:

| 글로벌 ChMS (현재) | AI Agentic Workflow (우리) |
|---------------------|--------------------------|
| 출석 데이터 조회 | 출석 감소 자동 감지 → 심방 제안 생성 |
| 봉사자 수동 스케줄링 | 은사·스케줄·이력 기반 최적 자동 배정 |
| 재정 보고서 수동 생성 | 월간 재정 보고서 자동 초안 + 이상 감지 |
| 방문자 정보 수집 | 방문자 등록 시 케어 파이프라인 자동 시작 |
| 이메일 템플릿 작성 | 개인화된 목회 서신 초안 자동 생성 |
| 주보 수동 제작 | 데이터 기반 주보 자동 생성 |

## 2. AI가 교회에서 할 수 있는 혁신적 기능

### 2.1 설교 준비 보조

**기능명**: 설교 리서치 & 아웃라인 자동화 (Sermon Research Assistant)

**설명**: 설교 주제/본문 구절 입력 시, AI가 관련 성경 구절, 신학적 주석, 역사·문화 배경, 예화 아이디어, 적용 포인트를 구조화된 아웃라인으로 생성한다.

**필요 데이터**:
- 설교 주제 또는 성경 본문 (사용자 입력)
- 이전 설교 아카이브 (중복 주제 방지용, `sermons/archive.yaml`)
- 절기 캘린더 (`calendar/events.yaml` — 대강절, 부활절 등 맥락 반영)

**AI가 하는 역할**:
- 입력 본문 기반 신학적 주석 요약 생성
- 관련 병행 구절 목록 (cross-reference) 추출
- 예화 아이디어 3-5개 제안
- 설교 구조 초안 (서론-본론-결론) 생성
- 대상 청중별 적용 포인트 차별화 (장년/청년/새신자)
- 이전 설교와의 중복 여부 경고

**사용자가 하는 역할**:
- 주제/본문 입력
- AI 초안 검토 및 신학적 검증
- 개인 경험/예화 추가
- 최종 설교 작성

**복잡도**: 중 (Bible knowledge base 구축 필요)

---

### 2.2 교인 이탈 위험 예측 (Churn Prediction)

**기능명**: 출석 위험 조기 경보 시스템 (Early Warning System)

**설명**: 출석 패턴, 헌금 패턴, 소그룹 참여 이력을 분석하여 이탈 위험 교인을 자동 감지하고, 담당 목회자에게 맞춤형 심방 및 연락 제안을 생성한다.

**필요 데이터**:
- `members/attendance.yaml` — 주별 출석 기록
- `members/giving.yaml` — 헌금 이력
- `groups/participation.yaml` — 소그룹 참여 이력
- `members/profile.yaml` — 교인 기본 정보

**AI가 하는 역할**:
- 3주 이상 결석 교인 자동 플래그 처리
- 헌금 급격 감소 교인 탐지 (전월 대비 50% 이상 감소)
- 소그룹 이탈 조합 패턴 감지 (출석↓ + 헌금↓ + 소그룹 결석 동시)
- 위험도 등급 산정 (HIGH/MEDIUM/LOW)
- 각 교인별 맞춤 심방 제안 메시지 초안 생성

**사용자가 하는 역할**:
- 주간 위험 교인 목록 리뷰
- 심방/연락 여부 결정
- 연락 메시지 개인화 후 발송 승인

**복잡도**: 중 (패턴 인식 로직 설계)

---

### 2.3 기도 요청 정리 및 추적

**기능명**: 기도 제목 관리 자동화 (Prayer Request Management)

**설명**: 교인이 제출한 기도 제목을 자동 분류(건강/가정/재정/진로/감사)하고, 주간 기도회 교재용 기도 목록을 자동 생성하며, 응답된 기도 제목을 추적 기록한다.

**필요 데이터**:
- `prayer/requests.yaml` — 기도 요청 원문
- `members/profile.yaml` — 요청자 정보
- `calendar/events.yaml` — 기도회 일정

**AI가 하는 역할**:
- 기도 제목 카테고리 자동 분류 (건강/가정/재정/진로/선교/감사/긴급)
- 긴급 기도 제목 자동 알림 (입원, 사망, 위기 키워드 감지)
- 주간 기도회용 기도 목록 자동 편집 (익명 처리 옵션)
- 기도 제목 응답 추적 기록 (교인 피드백 기반)
- 월간 기도 통계 보고서 생성 (주제별 분포, 응답률)

**사용자가 하는 역할**:
- 기도 제목 제출 (교인)
- 분류 결과 검토 및 게시 승인 (담당자)
- 응답된 기도 제목 표시 (교인/담당자)

**복잡도**: 낮 (분류 로직 단순)

---

### 2.4 교회 연간 보고서 자동 생성

**기능명**: 연간 결산 보고서 자동화 (Annual Report Generator)

**설명**: 연간 출석, 헌금, 사역 데이터를 종합하여 당회/제직회/공동의회 제출용 연간 보고서 초안을 자동 생성한다. 그래프, 통계표, 서술형 요약을 포함한다.

**필요 데이터**:
- `members/attendance.yaml` — 연간 출석 데이터
- `finance/ledger.yaml` — 연간 재정 데이터
- `events/log.yaml` — 연간 행사 기록
- `groups/stats.yaml` — 소그룹 현황
- `members/changes.yaml` — 등록/이명/소천 등 교인 변동

**AI가 하는 역할**:
- 연간 주요 통계 추출 및 시각화 (텍스트 기반 차트)
- 전년 대비 성장/감소 분석 서술
- 주요 사역 성과 요약 (AI가 events 데이터에서 추출)
- 차년도 사역 방향 제안 초안
- 교단 총회 보고서 양식에 맞춰 데이터 자동 배치

**사용자가 하는 역할**:
- 보고서 초안 검토 및 수정
- 목회적 방향성 반영 및 최종 승인
- 공동의회 제출

**복잡도**: 중 (다수 데이터 소스 통합)

---

### 2.5 교회 소식지/뉴스레터 자동 생성

**기능명**: 월간 교회 소식지 자동화 (Church Newsletter Generator)

**설명**: 당월 행사 기록, 새신자 환영, 헌신 이야기, 선교 소식을 취합하여 교인에게 발송할 월간 소식지 초안을 자동 생성한다.

**필요 데이터**:
- `events/log.yaml` — 당월 행사 기록
- `members/newcomers.yaml` — 신규 등록 교인
- `missions/updates.yaml` — 선교 소식
- `sermons/archive.yaml` — 설교 주제 요약

**AI가 하는 역할**:
- 당월 주요 행사 하이라이트 자동 선별 및 요약
- 새신자 환영 섹션 자동 생성 (개인 정보 비공개 옵션)
- 설교 주제 한 줄 요약
- 다음 달 주요 일정 미리 보기 자동 생성
- 기도 제목 목록 포함

**사용자가 하는 역할**:
- 특별 기고 글 작성
- 이미지/사진 선택
- 최종 편집 및 발송 승인

**복잡도**: 낮-중

### 2.6 봉사자 최적 배정 자동화

**기능명**: 봉사자 스케줄 최적화 엔진 (Volunteer Scheduling Optimizer)

**설명**: 교인의 봉사 가능 시간, 은사, 이전 봉사 이력, 팀 구성 규칙을 고려하여 주간/월간 봉사 스케줄을 자동 생성하고 알림 발송까지 처리한다.

**필요 데이터**:
- `volunteers/profiles.yaml` — 봉사자 은사·가능 시간 등록
- `volunteers/history.yaml` — 봉사 이력
- `services/schedule.yaml` — 예배 순서·봉사 항목
- `volunteers/constraints.yaml` — 팀 구성 규칙 (동일 가족 동일 팀 배제 등)

**AI가 하는 역할**:
- 은사·가능 시간 기반 최적 봉사자 자동 매칭
- 특정 교인 집중 배정 방지 (형평성 로직)
- 봉사 확인 요청 메시지 자동 생성
- 배정 거부 시 대체 후보 자동 제시
- 월간 봉사 참여율 보고서 생성

**사용자가 하는 역할**:
- 봉사 가능 시간·은사 등록 (교인)
- 스케줄 최종 확정 (봉사 부장)
- 특수 상황 수동 조정

**복잡도**: 중-높 (최적화 알고리즘 설계)

### 2.7 예배 순서지 자동 생성

**기능명**: 주보 예배 순서 자동화 (Worship Order Generator)

**설명**: 설교 주제, 찬양 목록, 봉사자 명단, 특별 순서를 데이터에서 자동 조합하여 예배 순서지 초안을 생성한다. (이미 MVP에 포함된 주보 자동생성의 예배순서 특화 모듈)

**필요 데이터**:
- `services/plan.yaml` — 이번 주 예배 계획
- `worship/songs.yaml` — 찬양 목록 및 가사
- `volunteers/assignments.yaml` — 봉사자 배정
- `sermons/this-week.yaml` — 설교 정보

**AI가 하는 역할**:
- 예배 시작~마침 전체 순서 자동 구성
- 순서별 담당자 자동 배치
- 찬양 가사 및 코드 자동 포함 (선택)
- PDF 출력 포맷 자동 생성

**사용자가 하는 역할**:
- 찬양 목록·설교 정보 입력
- 순서 검토 및 최종 승인

**복잡도**: 낮 (MVP 주보 자동생성과 통합)

### 2.8 교회 예산 계획 지원

**기능명**: 차년도 예산 계획 자동 초안 (Budget Planning Assistant)

**설명**: 전년도 실제 지출 데이터와 성장 목표를 기반으로 차년도 부서별 예산 초안을 자동 생성하고, 시나리오별 시뮬레이션을 제공한다.

**필요 데이터**:
- `finance/ledger.yaml` — 최근 3년 재정 데이터
- `events/plan.yaml` — 차년도 계획 행사
- `members/growth-target.yaml` — 성장 목표 수치

**AI가 하는 역할**:
- 전년 대비 비목별 증감 분석
- 부서별 예산 배분 초안 생성 (헌금 수입 대비 비율 기반)
- 고정비/변동비 분류 자동화
- 3가지 시나리오 (보수/기준/성장) 동시 생성
- 리스크 항목 자동 표시 (전년 대비 급증 비목)

**사용자가 하는 역할**:
- 사역 우선순위 지시 (어느 부서 강화)
- 시나리오 선택 및 조정
- 최종 예산안 승인 (당회/제직회)

**복잡도**: 중

### 2.9 절기별 행사 기획 보조

**기능명**: 절기 행사 기획 자동화 (Liturgical Season Planner)

**설명**: 부활절, 성탄절, 추수감사절 등 교회 절기에 맞는 예배 주제, 특별 행사 아이디어, 준비 체크리스트, 섭외 일정을 자동 생성한다.

**필요 데이터**:
- `calendar/liturgical.yaml` — 교회력 절기 정보
- `events/history.yaml` — 이전 절기 행사 기록
- `volunteers/availability.yaml` — 봉사자 가용성

**AI가 하는 역할**:
- 절기 6~8주 전 자동 기획 킥오프 알림
- 절기 특화 예배 주제 및 설교 시리즈 제안
- 세부 준비 체크리스트 자동 생성 (D-6주~D-Day)

## 3. 한국 교회 특수 기능

### 3.1 교단 총회 보고서 자동 생성

**기능명**: 교단 보고서 양식 자동화 (Denominational Report Automation)

**설명**: 대한예수교장로회(합동/통합), 기독교대한감리회 등 교단별 총회 보고서 양식에 맞춰 교세 현황, 재정 현황, 사역 현황 데이터를 자동 배치하여 보고서 초안을 생성한다.

**배경**: 한국 주요 교단(예장합동 11,788개, 예장통합 9,446개 교회)은 매년 총회에 정형화된 보고서를 제출해야 한다. 양식은 교단별로 상이하며, 데이터 수집 및 입력이 수 시간의 수동 작업을 요구한다.

**필요 데이터**:
- `members/census.yaml` — 교인 현황 (세례 교인, 학습 교인, 출석 교인 구분)
- `finance/annual-summary.yaml` — 재정 연간 요약
- `education/annual-summary.yaml` — 부서별 현황
- `denomination/templates/` — 교단별 양식 파일

**AI가 하는 역할**:
- 교단 보고서 필수 항목 자동 추출 및 배치
- 전년 대비 증감 수치 자동 계산
- 양식 항목과 내부 데이터 자동 매핑
- 미입력 항목 자동 감지 및 경고

**사용자가 하는 역할**:
- 교단 양식 등록 (최초 1회)
- 초안 검토 및 서명
- 총회 제출

**복잡도**: 중 (교단별 양식 다양성)

### 3.2 기부금 영수증 발행 관리

**기능명**: 연말 기부금 영수증 자동 생성 (Tax Receipt Automation)

**설명**: 교인별 연간 헌금 합산 내역을 자동 계산하고, 국세청 홈택스 호환 기부금 영수증 양식으로 자동 생성하며, 교인별 개인 수령 안내를 처리한다.

**배경**: 한국 세법상 교회는 교인 요청 시 기부금 영수증을 의무 발행해야 한다(소득세법 시행령 §80①5호). 영수증에는 기부자 성명, 주민등록번호, 기부금액, 기부 기간, 교회명, 사업자등록번호 등이 정확히 기재되어야 한다.

**필요 데이터**:
- `finance/giving.yaml` — 교인별 월별 헌금 이력
- `members/profile.yaml` — 교인 개인정보 (성명, 주민번호)
- `church/registration.yaml` — 교회 사업자등록 정보

**AI가 하는 역할**:
- 교인별 연간 헌금 합산 자동 계산
- 국세청 양식 기준 영수증 초안 자동 생성
- 발행 대상자 목록 자동 정리
- 발행 완료/미완료 추적 관리
- 발행 안내 메시지 자동 작성

**사용자가 하는 역할**:
- 개인정보 정확성 최종 확인
- 영수증 발행 승인
- 교인에게 수령 안내

**복잡도**: 낮-중 (세법 준수 검증 필요)

### 3.3 심방 기록 및 스케줄 관리

**기능명**: 심방 관리 자동화 (Pastoral Visitation Manager)

**설명**: 심방 필요 교인(입원, 장기 결석, 특별 사정)을 자동 감지하고, 담당 목회자/권사의 심방 스케줄을 제안하며, 심방 기록을 체계적으로 보관한다.

**배경**: 한국 교회에서 심방(가정 방문)은 핵심 목회 활동이다. 권사의 직무는 당회 지도하에 교인을 심방하는 것이며, 담임 목사도 정기 심방을 수행한다.

**필요 데이터**:
- `members/attendance.yaml` — 출석 현황 (결석 감지용)
- `members/care-notes.yaml` — 교인 특별 사정 기록
- `members/hospital.yaml` — 입원/병원 기록
- `staff/schedule.yaml` — 목회자/권사 일정

**AI가 하는 역할**:
- 심방 필요 교인 자동 분류 (입원/결석/위기/정기)
- 담당 목회자/권사 매칭 (구역 기반 자동 배정)
- 이동 경로 최적화 제안 (동선 효율화)
- 심방 후속 기도 제목 및 필요 사항 기록 보관
- 월간 심방 통계 보고서 생성

**사용자가 하는 역할**:
- 심방 스케줄 최종 확정
- 심방 내용 기록 입력
- 특별 케어 필요 사항 판단

**복잡도**: 낮-중

### 3.4 구역/셀/목장 편성 관리

**기능명**: 소그룹 편성 최적화 (Small Group Formation Optimizer)

**설명**: 교인의 거주지, 연령대, 신앙 성숙도, 가족 구성을 기반으로 구역/셀/목장 편성을 최적화하고, 분기별 재편성 제안을 자동 생성한다.

**배경**: 구역(지역 기반), 셀(기능 기반), 목장(관계 기반) 등 소그룹 모델은 한국 교회의 핵심 목회 구조이다.

**필요 데이터**:
- `members/profile.yaml` — 거주지, 연령, 가족 정보
- `groups/current.yaml` — 현재 소그룹 편성
- `groups/leaders.yaml` — 소그룹 리더 명단

**AI가 하는 역할**:
- 거주지 기반 지리적 클러스터링
- 연령/생애주기 유사성 기반 그룹 제안
- 새신자 배치 최적 그룹 자동 추천
- 소그룹 규모 불균형 자동 감지 (인원 초과/미달)
- 분기별 재편성 필요성 리포트

**사용자가 하는 역할**:
- 편성 원칙 정의 (거주지 우선 vs 관계 우선)
- 특별 배려 교인 수동 조정
- 최종 편성 확정 및 공지

**복잡도**: 중

## 4. 데이터 분석 & 인사이트 기능

### 4.1 출석 트렌드 분석

**기능명**: 출석 트렌드 다차원 분석 (Attendance Trend Analytics)

**설명**: 연도별/월별/절기별/시간대별 출석 추세를 자동 분석하여 교회 성장 패턴 및 이탈 패턴을 인사이트화한다.

**분석 항목**:
- 월별 출석률 추세 (이동 평균 포함)
- 절기별 출석 패턴 (성탄절, 부활절 스파이크 분석)
- 예배 시간대별 비율 (1부/2부/3부)
- 연령대별 출석률 비교
- 신규 등록 대비 정착률 (3개월/6개월/1년 코호트)
- 날씨/계절 요인 상관관계 (선택적)

**AI가 하는 역할**:
- 데이터 자동 집계 및 추세 계산
- 이상 패턴 자동 감지 (급격한 감소 경보)
- 전년 동기 대비 성장/감소 자동 계산
- 인사이트 서술 자동 생성 ("지난 6개월 2부 예배 출석이 23% 증가했습니다")

**복잡도**: 중

---

### 4.2 헌금 패턴 분석

**기능명**: 재정 이상 탐지 및 패턴 분석 (Financial Pattern Analytics)

**설명**: 헌금 수입의 정상 패턴을 학습하고, 이상 패턴(급격한 감소, 특정 헌금 항목 이상 증가)을 자동 감지하며, 예산 대비 실적을 실시간 추적한다.

**분석 항목**:
- 주별/월별 헌금 총액 추세
- 헌금 항목별 비율 변화 (주일헌금/감사헌금/선교헌금)
- 예산 대비 실제 수입 진행률
- 헌금 교인 수 추세 (총액 유지되어도 교인 수 감소 감지)
- 개인별 헌금 패턴 변화 (이탈 징후 감지용)

**AI가 하는 역할**:
- 헌금 통계 자동 집계
- 이상치 자동 탐지 및 알림
- 예산 집행률 경보 (초과 예고, 미집행 경고)
- 분기별 재정 건전성 리포트 자동 생성

**복잡도**: 중

### 4.3 교인 인구 분포 분석

**기능명**: 교인 통계 대시보드 (Membership Demographics Dashboard)

**설명**: 교인의 연령, 성별, 거주지, 직업, 신앙 연수 분포를 정기적으로 분석하여 교회 인구 구조 변화를 추적한다.

**분석 항목**:

- 거주지역 분포 (원거리 교인 파악)

**복잡도**: 낮

### 4.4 새신자 정착률 추적

**기능명**: 새신자 코호트 분석 (Newcomer Cohort Analysis)

**설명**: 등록 월별 새신자 코호트를 추적하여 3개월/6개월/1년 정착률을 자동 계산하고, 어느 단계에서 이탈이 많은지 분석한다.

**분석 항목**:
- 코호트별 1개월/3개월/6개월/1년 잔존율

- 케어 이벤트(환영 방문, 소그룹 연결)와 정착률 상관관계
- 소개자(기존 교인)별 새신자 정착률 비교

**AI가 하는 역할**:
- 코호트 자동 구성 및 추적
- 정착률 자동 계산
- 이탈 패턴 인사이트 생성
- 고위험 시점 사전 알림 (등록 후 6주차 결석 시)

**복잡도**: 중

### 4.5 봉사 참여율 분석

**분석 항목**:

- 봉사 영역별 인력 부족 자동 감지
- 신규 봉사자 유입률

**복잡도**: 낮-중

## 5. 워크플로우 자동화 아이디어 (정기 실행)

### 5.1 주간 자동화 (매주 월요일)

```yaml
weekly-monday-workflow:
  trigger: "매주 월요일 오전 9시"
  tasks:
    - 전주 출석 집계 및 결석 교인 목록 생성
    - 3주 이상 연속 결석 교인 → 심방 제안 목록 생성
    - 이번 주 봉사자 명단 확인 및 미확인자 알림 초안 생성
    - 이번 주 방문 예정 행사/심방 스케줄 요약
    - 주보 초안 생성 (설교 정보 + 주요 공지)
```

### 5.2 월간 자동화 (매월 1일)

```yaml
monthly-first-day-workflow:
  trigger: "매월 1일 오전 7시"
  tasks:
    - 전월 재정 보고서 자동 초안 생성
    - 전월 출석 통계 보고서 생성
    - 전월 새신자 현황 및 정착 현황 보고서
    - 이달 절기/특별 행사 30일 사전 알림
    - 기부금 영수증 발행 대기 목록 업데이트 (연말만)
    - 선교사 지원금 지급 예정 목록 확인
```

### 5.3 분기별 자동화

```yaml
quarterly-workflow:
  trigger: "분기 마지막 월 15일"
  tasks:
    - 교단 분기 보고서 초안 생성
    - 분기 재정 검토 보고서 (예산 대비 실적)
    - 소그룹 재편성 필요성 리포트 생성
    - 봉사자 번아웃 위험군 분석 보고서
    - 새신자 6개월 정착 현황 코호트 분석
```

### 5.4 연말 자동화

```yaml
year-end-workflow:
  trigger: "12월 1일 ~ 31일"
  tasks:
    - 연간 교세 현황 보고서 초안 생성 (공동의회용)
    - 연간 재정 결산 보고서 초안
    - 기부금 영수증 전체 교인 자동 생성
    - 차년도 예산안 초안 (시나리오 3가지)
    - 연간 사역 보고서 초안
    - 내년 1월~12월 절기 행사 캘린더 자동 생성
```

### 5.5 이벤트 트리거 자동화

```yaml
event-triggered-workflows:
  new-visitor:
    trigger: "새신자 등록 완료 시"
    tasks:
      - 환영 메시지 초안 생성 (담임 목사 명의)
      - 2주 후 재방문 확인 알림 스케줄
      - 소그룹 추천 (연령/지역 기반)
      - 새신자 담당 교인 자동 매칭 제안

  hospital-admission:
    trigger: "입원 정보 등록 시"
    tasks:
      - 담당 목회자 즉시 알림
      - 심방 스케줄 제안
      - 기도 제목 등록
      - 구역장/목자 알림

  member-marriage:
    trigger: "결혼 날짜 등록 시"
    tasks:
      - 결혼 준비 체크리스트 자동 생성
      - 봉사자 배정 제안
      - 결혼 감사헌금 수납 안내 초안

  member-deceased:
    trigger: "소천 정보 등록 시"
    tasks:
      - 부고 통지 초안 생성
      - 장례 봉사자 배정 제안
      - 유족 돌봄 스케줄 제안
      - 교적 처리 자동 안내
```

| T3-05 | **자연어 교회 행정 질의응답** | "지난 3개월 결석 5회 이상인 30대 교인 목록"을 자연어로 조회 | 전체 YAML 데이터 + 자연어→쿼리 변환 | NL 파싱, 데이터 조회, 결과 포맷팅 | 질문 입력, 결과 활용 | 높 |
| T3-06 | **예배 영상/음향 자동 최적화 제안** | 예배 녹화 분석 기반 음향 균형, 조명 타이밍 최적화 제안 | 영상 데이터 (비파일 기반) + 예배 피드백 | 영상 분석, 개선 제안 | 기술 담당자 적용 | 매우 높 |
| T3-07 | **교인 은사 발견 자동화** | 설문 응답 + 봉사 이력 + AI 인터뷰 기반 은사 프로파일 생성 | `volunteers/profiles.yaml` + 은사 설문 + 봉사 이력 | 은사 패턴 분석, 프로파일 생성, 봉사 매칭 | 인터뷰, 결과 확인 | 높 |
| T3-08 | **지역사회 영향력 분석** | 교인 거주 지역 대비 지역사회 인구 분석 + 전략적 사역 지역 제안 | `members/profile.yaml` (거주지) + 외부 인구 데이터 | 지리적 분석, 기회 지역 파악 | 선교 전략 결정 | 높 |
| T3-09 | **교회 사역 건강성 종합 점수 (ChMS Dashboard)** | 출석/재정/봉사/소그룹/새신자 5개 차원 건강성 점수 통합 대시보드 | 전체 데이터 통합 | 다차원 점수 산정, 약점 진단, 개선 제안 | 점수 검토, 전략 수립 | 높 |
| T3-10 | **카카오 MCP 통합 알림 자동화** | 기존 시스템과 카카오채널/카카오톡 연동으로 교인 실시간 소통 자동화 | 카카오 API + 내부 YAML 데이터 | 메시지 생성, 타이밍 최적화, 개인화 | 발송 승인, 특별 케이스 처리 | 높 (외부 API 연동) |
