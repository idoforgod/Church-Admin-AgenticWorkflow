### 1.2 스캔-앤-복제 파이프라인 요약

스캔-앤-복제 시스템(F-06)은 모든 문서 유형에 대해 4단계로 운영된다:

```
Stage 1 — 이미지/PDF 업로드
  inbox/templates/{category}-sample.{jpg|pdf}

Stage 2 — Claude 멀티모달 분석
  고정 영역 감지 (앵커: 교회명, 로고, 인장, 교단 헤더)
  가변 영역 감지 (데이터 슬롯: 날짜, 내용, 수신인, 금액)
  레이아웃 추출 (격자, 열, 글꼴 영역)

Stage 3 — 템플릿 생성
  {category}-template.yaml  (기계 가독 구조)
  {category}-output.md      (인간 가독 출력 형식)

Stage 4 — 인간 확인 + 반복 생성
  첫 실행: 인간이 구조 및 가변 슬롯 검증
  이후: YAML 소스에서 데이터 주입으로 완전 자동
```

### 1.3 핵심 설계 결정사항

1. **고정 vs 가변 분류**: 고정 영역은 여러 샘플 스캔 간 일관성으로 감지된다. 동일한 텍스트/이미지가 3개 이상 샘플에서 동일 위치에 나타나면 고정으로 분류된다.
2. **슬롯 타입 지정**: 모든 가변 슬롯은 타입화되어 있다(string, date, currency, integer, list, boolean). 생성 시 P1 Compliance 검증을 가능하게 한다.
3. **데이터 소스 추적성**: 모든 가변 슬롯은 정확한 YAML 파일 + 필드 경로를 지정하여 자동화된 채움과 검증을 가능하게 한다.
4. **교단 변형**: 템플릿은 3가지 주요 한국 개신교 교단의 교단별 분기를 포함한다: 예장통합(대한예수교장로회), 예장합동(대한예수교장로회합동), 기감(기독교대한감리회).

### 2.1 주보(주보 — 주간 교회 주보)

#### 개요

주보(bulletin)는 한국 교회에서 가장 자주 생산되는 문서다 — 매 주일 예외 없이 생성된다. 주간 정보 허브로 기능한다: 설교 세부사항, 예배 순서, 공지사항, 기도제목, 교인 축하. 일반적인 주보는 A4 크기(때로는 B5를 2배로 접어 4쪽 책자 형태)이며 교인 수 플러스 20% 버퍼 수량으로 인쇄된다.

주보는 가장 높은 스캔-앤-복제 ROI를 가지고 있다: 연간 52회 생산되고, 주간 구조는 매우 일관되며, 의미 있는 변경은 가변 데이터 슬롯에만 있다. 샘플 스캔에서 템플릿이 추출되면, 이후 생성은 수동 레이아웃 작업이 필요 없다.

#### 고정 영역

고정 영역은 날짜나 내용과 관계없이 주어진 교회의 모든 주보에서 동일하게 나타나는 영역이다.

| 영역 ID | 영역명 | 한국어 명칭 | 위치(일반적) | 내용 설명 |
|--------|-------|-----------|----------|---------|
| FR-BUL-01 | 교회명 배너 | 교회명 배너 | 상단 중앙, 전체 너비 | 대형 글꼴의 정식 교회명(바탕체 또는 굴림체, 24–36pt) |
| FR-BUL-02 | 교회 로고/십자가 기호 | 교회 로고/십자가 | 상단 좌측 또는 상단 중앙좌측 | 교회 로고 이미지 또는 양식화된 십자가. 교회마다 위치는 다르나 교회 내 일관됨 |
| FR-BUL-03 | 교단 헤더 | 교단 명칭 | 교회명 아래 또는 상단 우측 모서리 | 교단 소속 텍스트(예: "대한예수교장로회 [노회명]" 또는 "기독교대한감리회") |
| FR-BUL-04 | 주보 제목 라벨 | 주보 제목 라벨 | 교회명 아래 | "주 보"(정중함을 위해 띄어짐) 또는 "주간안내" 고정 활판 |
| FR-BUL-05 | 교회 주소 블록 | 교회 주소 블록 | 바닥글 영역 | 교회 거리 주소, 전화번호, 웹사이트, 이메일. 소형 글꼴(8–10pt)로 인쇄 |
| FR-BUL-06 | 장식 테두리/프레임 | 장식 테두리 | 바깥쪽 여백 | 전체 페이지 주변의 얇은 단선 또는 쌍선 프레임. 전통 교회에서 흔함 |
| FR-BUL-07 | 섹션 헤더 라벨 | 섹션 제목 | 좌측 여백 또는 인라인 | 고정 섹션 제목: "말씀"(말씀), "공지사항"(공지사항), "기도제목"(기도제목), "이번주 행사"(이번주 행사) |

#### 가변 영역

가변 영역은 각 주보 생성마다 변경되고 YAML 데이터 소스에서 채워진다.

| 영역 ID | 영역명 | 한국어 명칭 | 슬롯 타입 | 위치 | 비고 |
|--------|-------|-----------|---------|------|-----|
| VR-BUL-01 | 발행 번호 | 발행 번호 | integer | 주보 제목 근처 | 형식: "제 NNN호"(호 NNN) |
| VR-BUL-02 | 날짜 | 날짜 | date | 발행 번호 아래 | 형식: "2026년 3월 1일 주일" — 요일 "주일" 포함 |
| VR-BUL-03 | 설교 제목 | 설교 제목 | string | 눈에 띄게, 중앙 또는 상단 본문 | 종종 대형 글꼴, 때로는 박스 처리됨 |
| VR-BUL-04 | 성경 본문 | 성경 본문 | string | 설교 제목 인접 | 형식: "요한복음 3:16" 또는 "요 3:16" |
| VR-BUL-05 | 설교자명 | 설교자 | string | 성경 본문 아래 | 형식: "담임목사 [이름]" 또는 "[이름] 목사" |
| VR-BUL-06 | 설교 시리즈 | 설교 시리즈 | string (nullable) | 설교 제목 위 | 형식: "시리즈 N부: [시리즈명]" |
| VR-BUL-07 | 예배 순서 항목 | 예배 순서 | list[object] | 좌측 열 또는 중앙 | 번호 매긴 목록: 순서번호, 항목명, 담당자/찬송가번호 |
| VR-BUL-08 | 찬송가 번호 | 찬송가 번호 | list[integer] | 예배 순서 내 | 찬송가(한국 찬송가)의 번호 참조 |
| VR-BUL-09 | 공지사항 | 공지사항 | list[object] | 본문 섹션 | 각 항목: 제목 + 간단한 내용. 가변 개수(0–10개 항목) |
| VR-BUL-10 | 기도제목 | 기도 제목 | list[object] | 본문 섹션 | 각 항목: 분류 + 내용. 표준 분류: 교회, 국가, 선교, 교인 |
| VR-BUL-11 | 생일자 | 생일자 | list[string] | 작은 상자 또는 사이드바 | 형식: "○○○ 성도 (N월 N일)" — 부분 이름 개인정보 마스킹 흔함 |
| VR-BUL-12 | 결혼기념일 | 결혼기념일 | list[string] | 생일자와 동일 상자 | 형식: "○○○, ○○○ 집사 부부 (N월 N일)" |
| VR-BUL-13 | 이번 주 행사 | 이번 주 행사 | list[object] | 사이드바 또는 바닥글 섹션 | 날짜 + 행사명 + 시간 + 장소 |
| VR-BUL-14 | 다음 주 예고 | 다음 주 예고 | string (nullable) | 바닥글 영역 | 다음 주일 설교 또는 행사의 간단한 예고 |
| VR-BUL-15 | 헌금 봉사자 | 헌금 봉사자 | list[string] | 작은 사이드바 | 당주 헌금 집사/자원봉사자의 이름 |
| VR-BUL-16 | 출석 통계 | 출석 통계 | object (nullable) | 작은 표 또는 바닥글 | 지난주 예배별 및 총 출석 |

#### YAML 템플릿 스키마

```yaml
# bulletin-template.yaml
# Generated by template-scanner agent from inbox/templates/bulletin-sample.jpg
# Human-confirmed: [YYYY-MM-DD]

template_id: "bulletin-v1"
document_type: "bulletin"
version: "1.0"
church_name: "{{ church_state.church.name }}"
denomination: "{{ church_state.church.denomination }}"
scan_source: "inbox/templates/bulletin-sample.jpg"
confirmed_by: null  # set after human confirmation
confirmed_date: null

paper:
  size: "A4"              # A4 (210×297mm) most common; B5 for folded booklets
  orientation: "portrait"
  pages: 2                # 1 sheet front+back = 2 logical pages; 1 for single-sided
  folds: 0                # 0=flat, 1=half-fold (creates 4 logical pages from 2)

layout:
  margins:
    top_mm: 15
    bottom_mm: 15
    left_mm: 15
    right_mm: 15
  columns: 2              # most bulletins use 2-column layout
  column_gap_mm: 8

fixed_regions:
  - id: "FR-BUL-01"
    name: "church_name_banner"
    position: { top_mm: 5, left_mm: 0, width_pct: 100, height_mm: 20 }
    content: "{{ church_state.church.name }}"
    font: { family: "바탕체", size_pt: 28, weight: "bold", align: "center" }

  - id: "FR-BUL-02"
    name: "church_logo"
    position: { top_mm: 3, left_mm: 5, width_mm: 25, height_mm: 25 }
    content_type: "image"
    image_path: "assets/church-logo.png"

  - id: "FR-BUL-03"
    name: "denomination_header"
    position: { top_mm: 2, left_mm: 0, width_pct: 100, height_mm: 6 }
    content: "{{ church_state.church.denomination_full_name }}"
    font: { family: "굴림체", size_pt: 9, weight: "normal", align: "center" }

  - id: "FR-BUL-04"
    name: "bulletin_title_label"
    position: { top_mm: 26, left_mm: 0, width_pct: 100, height_mm: 10 }
    content: "주  보"
    font: { family: "바탕체", size_pt: 18, weight: "bold", align: "center" }

  - id: "FR-BUL-05"
    name: "church_address_block"
    position: { top_mm: 280, left_mm: 0, width_pct: 100, height_mm: 12 }
    content: "{{ church_state.church.address }} | TEL: {{ church_state.church.phone }}"
    font: { family: "굴림체", size_pt: 8, weight: "normal", align: "center" }

  - id: "FR-BUL-06"
    name: "decorative_border"
    type: "border"
    style: "single_line"
    margin_from_edge_mm: 3

variable_regions:
  - id: "VR-BUL-01"
    name: "issue_number"
    slot_type: "integer"
    position: { top_mm: 37, left_mm: 120, width_mm: 70, height_mm: 7 }
    format: "제 {value}호"
    data_source:
      file: "bulletin-data.yaml"
      field: "bulletin.issue_number"
    validation: { min: 1, monotonically_increasing: true }

  - id: "VR-BUL-02"
    name: "bulletin_date"
    slot_type: "date"
    position: { top_mm: 37, left_mm: 0, width_mm: 110, height_mm: 7 }
    format: "{year}년 {month}월 {day}일 주일"
    data_source:
      file: "bulletin-data.yaml"
      field: "bulletin.date"
    validation: { format: "YYYY-MM-DD", day_of_week: "sunday" }

  - id: "VR-BUL-03"
    name: "sermon_title"
    slot_type: "string"
    position: { top_mm: 50, left_mm: 0, width_pct: 100, height_mm: 12 }
    font: { family: "바탕체", size_pt: 16, weight: "bold", align: "center" }
    data_source:
      file: "bulletin-data.yaml"
      field: "bulletin.sermon.title"
    validation: { min_length: 1, max_length: 60 }

  - id: "VR-BUL-04"
    name: "scripture_reference"
    slot_type: "string"
    position: { top_mm: 63, left_mm: 0, width_pct: 100, height_mm: 7 }
    font: { family: "굴림체", size_pt: 12, align: "center" }
    data_source:
      file: "bulletin-data.yaml"
      field: "bulletin.sermon.scripture"
    validation: { min_length: 3, max_length: 40 }

  - id: "VR-BUL-05"
    name: "preacher_name"
    slot_type: "string"
    position: { top_mm: 71, left_mm: 0, width_pct: 100, height_mm: 7 }
    format: "설교 {value} 목사"
    data_source:
      file: "bulletin-data.yaml"
      field: "bulletin.sermon.preacher"

  - id: "VR-BUL-06"
    name: "sermon_series"
    slot_type: "string"
    nullable: true
    position: { top_mm: 45, left_mm: 0, width_pct: 100, height_mm: 6 }
    format: "[ {value} ]"
    data_source:
      file: "bulletin-data.yaml"
      field: "bulletin.sermon.series"

  - id: "VR-BUL-07"
    name: "worship_order"
    slot_type: "list[object]"
    position: { top_mm: 80, left_col: 0, width_col: 1, height_mm: 80 }
    item_height_mm: 8
    data_source:
      file: "bulletin-data.yaml"
      field: "bulletin.worship_order"
    item_format: "{order}. {item}  {detail}  {performer}"
    validation: { min_items: 3 }

  - id: "VR-BUL-09"
    name: "announcements"
    slot_type: "list[object]"
    position: { top_mm: 165, left_col: 0, width_col: 2, height_mm: 60 }
    data_source:
      file: "bulletin-data.yaml"
      field: "bulletin.announcements"
    item_format: "◆ {title}: {content}"
    filter: "priority != 'low' and expires >= bulletin.date"
    validation: { max_items: 8 }

  - id: "VR-BUL-10"
    name: "prayer_requests"
    slot_type: "list[object]"
    position: { top_mm: 230, left_col: 0, width_col: 2, height_mm: 40 }
    data_source:
      file: "bulletin-data.yaml"
      field: "bulletin.prayer_requests"
    item_format: "• [{category}] {content}"
    validation: { min_items: 1 }

  - id: "VR-BUL-11"
    name: "birthday_members"
    slot_type: "list[string]"
    position: { top_mm: 165, left_col: 1, width_mm: 80, height_mm: 25 }
    data_source:
      file: "members.yaml"
      field: "members[*]"
      filter: "birth_date matches this_week_month_day and status == 'active'"
      transform: "mask_name_middle_char(name) + ' 성도 (' + format(birth_date, 'M월 d일') + ')'"
    nullable: true

  - id: "VR-BUL-12"
    name: "wedding_anniversaries"
    slot_type: "list[string]"
    position: { top_mm: 192, left_col: 1, width_mm: 80, height_mm: 20 }
    data_source:
      file: "members.yaml"
      field: "members[*]"
      filter: "family.relation == 'household_head' and family.wedding_anniversary matches this_week_month_day"
      transform: "family_display_names + ' 부부 (' + format(anniversary, 'M월 d일') + ')'"
    nullable: true

  - id: "VR-BUL-15"
    name: "offering_team"
    slot_type: "list[string]"
    position: { top_mm: 215, left_col: 1, width_mm: 80, height_mm: 15 }
    data_source:
      file: "bulletin-data.yaml"
      field: "bulletin.offering_team"

output_format:
  type: "markdown"
  output_path_template: "bulletins/{date}-bulletin.md"
  pdf_ready: false  # PDF conversion deferred to Phase 2
```

#### 데이터 소스 매핑

| 가변 슬롯 | YAML 파일 | 필드 경로 | 변환 | 검증 |
|---------|----------|---------|------|-----|
| 발행 번호 | `bulletin-data.yaml` | `bulletin.issue_number` | "제 N호"로 형식화 | integer ≥ 1, 단조 증가 |
| 날짜 | `bulletin-data.yaml` | `bulletin.date` | "YYYY년 MM월 DD일 주일"로 형식화 | YYYY-MM-DD, 반드시 일요일 |

### 2.2 헌금 영수증(헌금 영수증 — 세금 기부금 영수증)

#### 개요

헌금 영수증(tax donation receipt)은 한국에서 법적으로 의미 있는 문서다. 소득세법 시행령 §80①5호에 따르면, 교회는 교인 요청 시 공식 기부금 영수증을 발급할 의무가 있으며, 교인들은 이를 연말 세금 공제에 사용한다. 중간 규모 교회(250명 교인)는 연간 100–200장의 영수증을 발급하며, 주로 연말 일괄 실행 기간(12월–1월)에 발생한다.

영수증은 특정 법적으로 필수인 필드를 포함해야 한다: 기부자 성명, 기부자 ID번호(또는 부분), 교회 등록번호, 기부금액(한국 원 및 한글 숫자 표기법), 기간, 그리고 교회 공식 인장(직인). 스캔-앤-복제 템플릿은 인장 위치를 고정 앵커로 정확하게 캡처해야 한다 — AI 생성은 인장 영역 위에 가변 콘텐츠를 배치하면 안 된다.

#### 고정 영역

| 영역 ID | 영역명 | 한국어 명칭 | 위치(일반적) | 내용 설명 |
|--------|-------|-----------|----------|---------|
| FR-RCP-01 | 영수증 제목 | 영수증 제목 | 상단 중앙 | "기부금영수증" 또는 "헌금영수증" 대형 글꼴(바탕체, 20–24pt) |
| FR-RCP-02 | 교회명 | 교회명(발행기관) | 상단 본문 | 발행 기관으로서의 정식 교회명 |
| FR-RCP-03 | 교회 등록번호 | 교회 등록번호 | 교회명 아래 | 정부 등록번호(사업자등록번호 또는 고유번호) |
| FR-RCP-04 | 교회 주소 | 교회 주소 | 등록번호 아래 | 교회의 완전한 우편 주소 |
| FR-RCP-05 | 대표자명 | 대표자명 | 발행인 블록 | 법적 대리인으로서의 담임목사 이름 |
| FR-RCP-06 | 인장 위치 영역 | 직인 위치 | 발행인 블록 우측 하단 | 공식 교회 인장(직인)을 위한 예약 영역. 이 영역에는 가변 콘텐츠가 없어야 함. |
| FR-RCP-07 | 영수증 유형 라벨 | 영수증 유형 라벨 | 헤더 | "종교단체"(종교단체) 분류 라벨 |
| FR-RCP-08 | 법적 근거 텍스트 | 법적 근거 문구 | 바닥글 | 소득세법 §34 및 시행령 §80을 언급하는 표준화된 법적 텍스트 |
| FR-RCP-09 | 일련번호 라벨 | 일련번호 라벨 | 헤더 우측 | 고정 라벨 "No." — 숫자 자체는 가변 |

## 3. 한국 교회 서식 관례

한국 교회 문서 서식 관례를 이해하는 것은 템플릿 스캐너가 영역을 고정 vs 가변으로 올바르게 분류하고, 템플릿 생성기가 문화적으로 그리고 기능적으로 올바른 산출물을 생성하기 위해 필수적이다.

### 3.1 용지 크기 관례

| 문서 유형 | 주 크기 | 보조 옵션 | 비고 |
|---------|-------|--------|-----|
| 주보(주보) | A4(210×297mm) | B5 접기(4쪽 책자), A5 | B5는 전통/형식적 교회에서 사용; A4가 가장 흔함 |
| 영수증(영수증) | A4 | 없음 | 법적 문서는 항상 A4 |
| 예배 순서지(순서지) | A5 또는 A4 접기 | A4의 절반 | 종종 주보 내부에 인쇄됨 |
| 공문(공문) | A4 | 없음 | 한국 행정 전통에서 A4로 표준화됨 |
| 회의록(회의록) | A4 | 없음 | 항상 A4; 다중 페이지 가능 |
| 증서(증서) | A4 | 맞춤 증서 용지 | 종종 무거운 용지(160–200gsm)에 인쇄됨 |
| 초청장(초청장) | A5 | DL 봉투 크기, 맞춤 | 형식적 초청장의 경우 더 작은 형식이 흔함 |

### 3.2 글꼴 관례

한국 교회 문서는 두 가지 주요 서체를 사용하며, 둘 다 표준 한국 사무 소프트웨어(HWP, MS Word 한글판)에 포함되어 있다:

| 서체 | 한국어명 | 특성 | 교회 문서의 전형적 사용 |
|------|--------|------|----------------------|
| **바탕체** | 바탕체 | 세리프, 전통, 정중. Ming/Song 서체와 유사. | 문서 제목, 설교 제목, 증서 본문, 공문. 형식적 문서. |
| **굴림체** | 굴림체 | 산세리프, 깔끔, 현대적. | 연락처, 주소, 캡션, 더 작은 본문. 정보 섹션. |
| **궁서체** | 궁서체 | 매우 정중, 서예적 세리프. | 매우 정중한 상황의 증서 제목. 표준 주보에서는 드뭄. |
| **돋움체** | 돋움체 | 산세리프, Gulim과 유사. | 부차 본문, 현대 스타일의 주보. |

**글꼴 크기 관례**:
- 문서 제목: 20–36pt(바탕체, 굵음)
- 주 제목: 14–18pt
- 본문: 10–12pt
- 캡션, 주소, 법적 텍스트: 8–9pt
- 증서 이름: 18–24pt(바탕체, 굵음)

### 3.3 인장(직인) 배치 규칙

직인(official seal/stamp)은 한국 교회 문서에서 가장 중요한 요소 중 하나다. 잘못된 배치나 부재는 문서의 공식적 지위를 무효화한다.

**인장 유형**:
- **직인(職印)**: 교회의 공식 기관 인장. 원형 또는 직사각형. 교회명 + 한글 印 문자 포함.
- **인감(印鑑)**: 권한 있는 서명자(목사)의 개인 인장. 작음, 원형.

**문서 유형별 배치 규칙**:

| 문서 유형 | 인장 유형 | 배치 영역 | 위치 설명 |
|---------|---------|---------|---------|
| 공문(공문) | 직인 | FR-LET-05: 발행인 블록 우측 하단 | 대표자 성명/서명선과 겹침, 서명 후 날인 |
| 회의록(회의록) | 직인 | FR-MIN-06: 서명 블록 하단 | 의장 서명선 위 또는 인접 날인 |
| 증서(세례증서/이명증서) | 직인 + 인감 | FR-CRT-06: 중앙/눈에 띄는 위치 | 본문 아래의 시각적으로 눈에 띄는 위치에 날인 |
| 영수증(영수증) | 직인 | FR-RCP-06: 발행인 블록 우측 하단 | 세금 기부금 영수증의 법적 유효성을 위해 필수 |
| 주보(주보) | 없음 | 해당 없음 | 주보는 인장이 필요하지 않음 |
| 예배 순서지(순서지) | 없음 | 해당 없음 | 인장 불필요 |

**템플릿 스캐너를 위한 중요한 규칙**: 템플릿 스캔 중 인장 영역을 감지할 때, 스캐너는:
1. 샘플 스캔에서 원형 또는 직사각형 인장 인상을 식별
2. 영역 좌표를 `reserved_seal_zone` 고정 영역으로 기록
3. 생성 중 가변 데이터가 그 영역에 배치되지 않도록 `guard: "NO_VARIABLE_CONTENT"`로 표시

### 3.4 교단 헤더 패턴

교단 헤더는 정중한 문서(공문, 증서, 회의록) 상단에 나타나며 교회가 활동하는 교회적 권위를 식별한다. 정확한 형식은 교단에 따라 다르다:

| 교단 | 한국어명 | 전형적 헤더 텍스트 | 약자 |
|----|--------|------------------|----|
| 예장통합 | 대한예수교장로회(통합) | "대한예수교장로회 [○○노회]" | PCK |
| 예장합동 | 대한예수교장로회(합동) | "대한예수교장로회합동 [○○노회]" | PROK |
| 기독교대한감리회 | 기독교대한감리회(기감) | "기독교대한감리회 [○○연회]" | KMC |
| 한국기독교장로회 | 한국기독교장로회(기장) | "한국기독교장로회 [○○노회]" | PCKK |
| 기독교대한성결교회 | 기독교대한성결교회 | "기독교대한성결교회 [○○지방회]" | KHC |
| 기독교한국루터회 | 기독교한국루터회 | "기독교한국루터회" | KLC |

**스캔-앤-복제 주목사항**: 교단 헤더는 첫 스캔에서 추출되면 모든 후속 문서에 대한 고정 영역이 된다. 스캐너는 교단별 노회(노회) 또는 지구(연회/지방회) 명칭을 포함하는 정확한 교단 문자열을 캡처해야 하며, 이는 교회의 공식 소속의 일부다.

### 3.5 세로쓰기 영역

일부 한국 교회 문서, 특히 증서 및 형식적 초청장은 장식 또는 헤더 섹션에서 세로쓰기(세로쓰기/종서)를 사용한다. 이는 동아시아 활판의 전통적 관례다.

**템플릿 스캐너를 위한 감지 규칙**:
- 세로쓰기 영역은 경계 상자에서 문자 높이 > 문자 너비로 식별됨
- 문자는 위에서 아래로, 우측에서 좌측으로 흐름
- 전형적 사용: 증서 제목, 정중한 헤딩, 매우 정중한 증서의 이름

**템플릿 표현**:
```yaml
text_direction: "vertical"    # "horizontal" (default) or "vertical"
vertical_flow: "top_to_bottom_right_to_left"   # traditional Korean vertical
```

**실용적 주목사항**: 본 시스템에서 사용하는 Markdown 출력 형식은 세로쓰기 렌더링을 기본적으로 지원하지 않는다. 세로쓰기 영역은 템플릿 YAML의 메타데이터로 캡처되지만 Markdown 출력에서는 가로 텍스트로 렌더링된다. PDF 변환(Phase 2)은 실제 세로쓰기 렌더링을 처리할 것이다.

### 3.6 경어체 및 정중한 언어 관례

한국 교회 문서는 전체적으로 정중한 레지스터(높임말)를 사용한다. 핵심 관례:

**회의록(회의록)의 경우**:
- 3인칭 보도식: "○○ 장로가 동의하고 ○○ 장로가 재청하여 가결하다."
- 수동태 선언식: "[의안]이 상정되어 논의 후 원안대로 통과되다."
- 축약형 없음, 격식적 조사 없음

**공문(공문)의 경우**:
- 개시: "귀 [수신처] 교회(기관)의 무궁한 발전을 기원합니다."
- 본문: 정중한 동사 종결(-습니다, -하겠습니다)
- 종결: "위와 같이 알려드립니다." 또는 "위와 같이 공문을 발송합니다."

**증서(증서)의 경우**:
- 현재형 선언식: "위 사람은 [year]년 [month]월 [day]일 본 교회에서 세례를 받았음을 증명합니다."
- 정중한 주장: 절대 격식적이지 않음

이러한 관례는 각 템플릿의 `style_guidelines`로 인코딩되며 가변 콘텐츠를 작성할 때 template-generator agent에서 참조된다.

## 4. 우선순위 분류

PRD §5.1 F-06 및 §8.1에 따라, 문서 유형은 3가지 구현 등급으로 분류된다.

### 4.1 Tier A — 즉시 구현(즉시 — M1 마일스톤)

**기준**: 높은 빈도(주간 또는 연간 일괄), 직접 M1 기능 요구사항, 시간 절감에 대한 최고 ROI.

| 문서 유형 | 빈도 | 연간 볼륨 | 인스턴스당 시간 절감 | M1 이유 |
|---------|------|---------|-------------------|--------|
| **주보(주보)** | 주간 | 연 52회 | ~2.5시간 | 핵심 MVP 기능(F-01). 연 52회 반복은 모든 문서 유형 중 템플릿 ROI가 가장 높음. |
| **헌금 영수증(영수증)** | 연간 일괄 | 연 100–200장 | ~25분/장 | 법적 의무 + 연말 일괄 생성(T1-03). 재정 담당 집사에게 높은 고통점. |
| **예배 순서지(순서지)** | 주간 | 연 52회 | ~30분 | 종종 주보와 결합됨; 연 52회 반복; 주보 데이터 소스와 밀접하게 결합. |

**Tier A 내 구현 우선순위**:
1. 주보(주보) — 가장 높은 빈도, 예배 순서지의 기초
2. 예배 순서지(순서지) — 주보와 데이터 소스의 80% 공유
3. 헌금 영수증(영수증) — 가장 높은 법적 의미, 연말에 일괄 생성 가능

### 4.2 Tier B — Phase 1 확장(M2 마일스톤)

**기준**: 중간 빈도(월간 또는 연 몇 회), 중요하나 일상 운영상 중요하지 않음, Tier A 인프라가 확립된 후 복잡성 관리 가능.

| 문서 유형 | 빈도 | 연간 볼륨 | M2 이유 |
|---------|------|---------|--------|
| **공문(공문)** | 월 2–5회 | 연 24–60장 | T1-05 기능. 확립된 교인 + 일정 데이터 사용. 증서보다 템플릿이 더 간단. |
| **회의록(당회/제직회 회의록)** | 월 1–4회 | 연 12–48장 | M1 인프라에서 이미 있는 교인 명부 데이터 필요. 통치 기록 준수 필요. |
| **증서(세례증서, 이명증서)** | 연 2–4회 | 연 2–20장 | T1-02 기능. 낮은 빈도이나 높은 법적 의미. M1 교인 관리 인프라에 의존. |

### 4.3 Tier C — Phase 2 / M3 연기

**기준**: 낮은 빈도, 높은 설계 가변성(표준화하기 더 어려움), 또는 M1/M2에서 아직 없는 기능에 의존.

| 문서 유형 | 빈도 | 연기 이유 |
|---------|------|---------|
| **교단 보고서(교단 보고서)** | 연간 | 높은 복잡성: 모든 데이터 소스 집계, 교단 본부가 명시한 형식, 수동 검증 필요. T2-03으로 연기. |
| **뉴스레터(교회 소식지)** | 월간 | 가변 설계, 혼합 콘텐츠 유형, 아직 PRD 범위 내의 스캔-앤-복제 대상이 아님. |
| **개인별 초청장(개인별 초청장)** | 임시 | 초청장 생성은 Tier C; 교인별 일괄 개인화된 초청장(교인당 1장)은 아직 설계되지 않은 일괄 루프 인프라 필요. |

**초청장(초청장)에 대한 주목**:
- 표준(단일) 초청장: 기존 schedule.yaml 데이터를 사용하는 Tier B 능력
- 일괄 개인화된 초청장(교인당 1장): Tier C, M2 일괄 인프라 필요
- PRD는 초청장을 F-06에서 Tier C로 나열; 이 분석은 일괄 개인화 모드에 동의

## 5. 교단 변형

한국 교회 문서 서식은 교단 간에 의미 있게 다르다. 스캔-앤-복제 시스템은 교단별 코드 변경 없이 교단별 변형을 수용해야 한다.

### 5.1 장로교(예장통합 / 예장합동)

두 가지 최대 장로교 교단은 유사한 통치 구조를 공유하나 일부 문서 관례에서 다르다.

| 측면 | 예장통합(PCK) | 예장합동(PROK) | 비고 |
|------|------------|------------|-----|
| 교단 헤더 | "대한예수교장로회 [노회명]" | "대한예수교장로회합동 [노회명]" | 정확히 일치해야 함 |
| 당회 권위 | 당회(당회)가 최고 | 동일 | |
| 회의록 제출 | 노회에 연간 | 노회에 연간 | 다른 노회 |
| 세례 형태 | 성인 + 영유아 | 성인 + 영유아 | 유사한 형식 |
| 이명 증서 | 당회가 발급 | 동일 | |
| 공문 번호 매김 | 순차 연간 | 순차 연간 | |
| 인장 요구 | 직인 필수 | 직인 필수 | |
| 공동의회 보고 | 연간 필수 | 연간 필수 | 다른 형태 템플릿 |

### 5.2 감리교(기독교대한감리회 — 기감)

감리회 구조는 통치 계층에서 장로교와 크게 다르다.

| 측면 | 기감(KMC) | 장로교 비교 |
|------|---------|----------|
| 교단 헤더 | "기독교대한감리회 [연회명]" | 노회 → 연회(지구 대회) |
| 통치기구 | 교구회(회로 회의) 당회 아님 | 다른 기구명 |
| 회의록 | 교구회 회의록 | 당회록 → 교구회록 |
| 이명 증서 | 이명증서(동일 명칭, 다른 권위) | 당회 아님 교구회가 발급 |
| 연간 보고 | 연회 보고서 | 장로교 노회와 다른 형태 |
| 세례 형태 | 장로교와 유사 | 약간의 문구 차이 |

### 5.3 템플릿 교단 매개변수

중복 템플릿 없이 교단 변형을 처리하려면, 각 템플릿은 조건부 렌더링을 운영하는 `denomination` 매개변수를 포함한다:

```yaml
# In each template:
denomination_config:
  source: "church-state.yaml"
  field: "church.denomination_code"
  # Values: "PCK", "PROK", "KMC", "PCKK", "KHC", etc.

conditional_content:
  - condition: "denomination_code in ['PCK', 'PROK']"
    overrides:
      governing_body_name: "당회"
      minutes_type: "당회록"
      presbytery_level: "노회"

  - condition: "denomination_code == 'KMC'"
    overrides:
      governing_body_name: "교구회"
      minutes_type: "교구회록"
      presbytery_level: "연회"
```

### 5.4 교단별 스캔 우선순위

초기 구현(M1)에서 스캔-앤-복제 시스템은 다음을 우선순위로 한다:
1. **예장통합(PCK)**: 최대 교단, 한국 개신교 교회의 ~30%
2. **예장합동(PROK)**: 두 번째 최대, ~25%
3. **기감(KMC)**: 세 번째 최대, ~15%

결합 범위: 3가지 교단 구성으로 한국 개신교 교회의 ~70%.

### 6.2 가변 슬롯 타입 시스템

파이프라인의 데이터 주입 엔진은 해당 검증으로 이 슬롯 유형을 지원해야 한다:

| 슬롯 유형 | YAML 표현 | 검증 규칙 | 변환 함수 |
|---------|----------|---------|---------|
| `string` | `str` | min_length, max_length, non_empty | mask_name, format_title |
| `integer` | `int` | min, max, monotonically_increasing | format_with_unit |
| `date` | `str` (YYYY-MM-DD) | valid date, day_of_week constraints | korean_date_format, korean_weekday |
| `time` | `str` (HH:MM) | valid time, 24h | korean_time_format (오전/오후) |
| `date_range` | `object` {start, end} | end > start | korean_date_range_format |
| `currency` | `int` (KRW) | > 0, integer (no cents) | krw_format (₩X,XXX), korean_numeral |
| `text` | `str` (multiline) | non_empty | line_wrap, honorific_check |
| `list[string]` | `list[str]` | min_items, max_items | join, numbered_list |
| `list[object]` | `list[dict]` | item schema validation | table_format, item_template |
| `enum` | `str` | allowed_values | display_name lookup |
