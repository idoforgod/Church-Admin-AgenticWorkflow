# 빠른 시작 가이드 — 30분 만에 첫 주보 만들기

이 가이드는 교회 행정 시스템을 사용하여 첫 주보를 만드는 전체 과정을 안내합니다. 데이터 입력부터 최종 주보 출력까지 30분 이내에 완료하는 것이 목표입니다. [trace:step-1:personas]

**대상 사용자**: 행정 간사 (첫 실행 시 IT 자원봉사자 도움 권장)
**사전 조건**: 설치 완료 ([설치 가이드](installation-guide.ko.md) 참조)

---

## 전체 흐름

```
Prepare Data (10 min) → Start Claude (2 min) → Generate Bulletin (5 min) → Review & Approve (10 min) → Done!
```

---

## 1단계: 이번 주 데이터 준비 (10분)

### 1.1 설교 정보 수정

텍스트 편집기에서 `data/bulletin-data.yaml`을 열고 설교 부분을 수정합니다:

```yaml
sermon:
  title: "은혜의 능력"           # This week's sermon title
  scripture: "에베소서 2:8-10"    # Bible passage
  pastor: "이성훈"               # Preaching pastor
```

### 1.2 공지사항 수정

같은 파일에서 공지사항 부분을 수정합니다:

```yaml
announcements:
  - title: "수요예배 안내"
    content: "이번 주 수요예배는 오후 7시 30분에 진행됩니다."
  - title: "교회 소풍"
    content: "5월 첫째 주 토요일 교회 소풍이 있습니다. 참가 신청서를 제출해주세요."
```

### 1.3 찬송가 번호 수정

```yaml
hymns:
  opening: 21         # Opening hymn number
  offertory: 94       # Offertory hymn number
  closing: 370        # Closing hymn number
```

### 1.4 파일 저장

`data/bulletin-data.yaml`을 저장합니다. 시스템이 자동으로 다음을 처리합니다:
- `data/members.yaml`에서 생일/기념일 해당 교인 추출
- `data/schedule.yaml`에서 이번 주 일정 반영
- `data/newcomers.yaml`에서 새신자 요약 생성

---

## 2단계: Claude Code 시작 (2분)

### 2.1 터미널 열기

터미널 애플리케이션을 엽니다 (IT 자원봉사자가 미리 설정해 놓았을 것입니다).

### 2.2 프로젝트 폴더로 이동

```bash
cd ~/Documents/church-admin-system/church-admin
```

### 2.3 Claude Code 시작

```bash
claude
```

Claude가 초기화될 때까지 기다립니다. 시스템이 준비되었음을 알리는 프롬프트가 표시됩니다.

---

## 3단계: 주보 생성 (5분)

### 3.1 명령 입력

한국어로 입력합니다:

```
이번 주 주보 만들어줘
```

Claude가 다음을 수행합니다:
1. `data/bulletin-data.yaml`에서 설교 정보, 공지사항, 찬송가 읽기
2. `data/members.yaml`에서 이번 주 생일 및 기념일 교인 확인
3. `data/schedule.yaml`에서 예배 시간 및 특별 행사 확인
4. `data/newcomers.yaml`에서 새신자 수 확인
5. 16개 가변 영역(VR-BUL-01~VR-BUL-16) 조합
6. 주보 템플릿(`templates/bulletin-template.yaml`) 적용
7. 주보 마크다운 파일 생성

### 3.2 생성 대기

처리에 1~3분이 소요됩니다. Claude가 각 섹션을 조합하는 과정을 표시합니다.

---

## 4단계: 검토 및 승인 (10분)

### 4.1 출력물 검토

Claude가 생성된 주보를 검토용으로 보여줍니다. 다음 항목을 확인하세요:

- [ ] 설교 제목과 성경 구절이 정확한지
- [ ] 목사님 이름이 맞는지
- [ ] 찬송가 번호가 선택한 것과 일치하는지
- [ ] 공지사항이 정확하고 빠짐없는지
- [ ] 생일/기념일 교인이 맞는지 (기록과 대조)
- [ ] 예배 시간이 정확한지
- [ ] 특별 행사가 나열되어 있는지
- [ ] 새신자 환영 부분이 적절한지

### 4.2 수정 요청 (필요 시)

수정이 필요하면 한국어로 Claude에게 말합니다:

```
설교 제목을 "새 생명의 약속"으로 변경해줘
```

또는

```
공지사항에 "성가대 연습 시간 변경: 토요일 오후 3시" 추가해줘
```

### 4.3 주보 승인

모든 내용이 확인되면:

```
주보 승인
```

주보가 타임스탬프와 함께 `bulletins/` 디렉터리에 저장됩니다.

---

## 5단계: 출력물 확인 (3분)

### 5.1 주보 파일 찾기

생성된 주보는 마크다운 파일로 저장됩니다:

```bash
ls bulletins/
# bulletin-2026-03-01.md (example)
```

### 5.2 인쇄용 변환 (선택 사항)

PDF나 Word로 변환이 필요한 경우:
- `.md` 파일을 아무 마크다운 편집기에서 열기
- PDF로 내보내 인쇄
- 또는 교회의 기존 워드 프로세서 템플릿에 복사하여 붙여넣기

---

## 축하합니다!

첫 주보를 성공적으로 만들었습니다. 전체 과정이 약 20~30분 정도 걸렸을 것입니다.

### 다음 주부터는 어떻게 되나요?

사용할수록 점점 빨라집니다:

| 주차 | 예상 소요 시간 | 이유 |
|------|--------------|------|
| 첫 주보 | 30분 | 시스템 익히기 |
| 두 번째 주보 | 15분 | 데이터 입력에 익숙해짐 |
| 세 번째 이후 | 10분 | 변경된 항목만 수정하면 됨 |

### 매주 반복하는 작업 흐름

매주 다음만 하면 됩니다:
1. `data/bulletin-data.yaml`에서 설교 정보와 공지사항 수정 (5분)
2. `이번 주 주보 만들어줘` 실행 (2분)
3. 검토 및 승인 (3분)

나머지는 시스템이 자동으로 처리합니다 — 생일, 일정, 새신자 현황, 서식 적용까지 모두 자동입니다.
