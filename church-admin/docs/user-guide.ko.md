# 사용자 가이드 — 행정 간사를 위한 일상 업무 안내

교회 행정 AI 시스템을 사용하여 일별, 주별, 월별 업무를 처리하는 방법을 안내하는 종합 가이드입니다. CLI 경험이 전혀 없는 비기술 사용자를 위해 작성되었습니다. [trace:step-1:personas]

**대상 사용자**: 행정 간사 (김미영 페르소나: 한글/엑셀 능숙, CLI 경험 없음)
**IT 지원**: 교회 IT 자원봉사자가 설치 및 유지보수를 담당합니다

---

## 시스템 작동 방식

교회 행정 시스템은 AI 에이전트를 활용하여 반복적인 업무를 자동화합니다. 다음과 같은 방식으로 시스템과 소통합니다:

1. **파일 넣기** — `inbox/` 폴더에 파일(Excel, Word, 이미지) 넣기
2. **한국어 명령 입력** — 자연어로 명령하기
3. **AI 결과물 검토 및 승인** — 생성된 결과물 확인하기

AI가 복잡한 작업을 처리하고, 최종 결정은 사용자가 내립니다.

---

## 시스템 시작하기

IT 자원봉사자가 터미널을 열고 시스템을 시작하는 방법을 알려줄 것입니다:

```bash
cd ~/Documents/church-admin-system/church-admin
claude
```

시작되면 한국어로 명령을 입력할 수 있습니다.

## 주간 업무

### 업무 1: 주보 제작

**시기**: 매주 목요일 또는 금요일
**소요 시간**: 10~15분

#### 1단계: 주보 데이터 수정

`data/bulletin-data.yaml`을 열어 다음을 수정합니다:
- 설교 제목과 성경 구절
- 공지사항 (새 항목 추가, 지난 항목 삭제)
- 찬송가 번호
- 특별 기도 요청

#### 2단계: 생성

```
이번 주 주보 만들어줘
```

#### 3단계: 검토

AI가 주보를 보여줍니다. 각 항목을 확인하세요:
- 설교 정보
- 예배 순서
- 공지사항
- 생일 및 기념일 (교적에서 자동 추출)
- 이번 주 일정

#### 4단계: 승인 또는 수정

- 이상 없으면: "주보 승인"
- 수정이 필요하면: AI에게 변경 내용을 알려주세요 (예: "공지사항 두번째 항목 삭제해줘")

---

### 업무 2: 새신자 관리

**시기**: 매주 월요일
**소요 시간**: 5~10분

#### 새신자 현황 확인

```
새신자 현황 보여줘
```

AI가 다음을 보여줍니다:
- 이번 주 첫 방문자
- 후속 관리가 필요한 새신자 (2주차 점검)
- 단계 전환이 가능한 새신자

#### 새 방문자 등록

주일에 새로운 분이 방문했다면:

```
새신자 등록
```

AI가 다음 정보를 요청합니다:
- 이름
- 전화번호
- 방문 경로
- 참석 예배

#### 후속 연락이 지연된 새가족 확인

```
재방문 확인 필요한 새가족 보여줘
```

---

### 업무 3: 일정 관리

**시기**: 수시
**소요 시간**: 변경 건당 2~5분

#### 이번 주 일정 보기

```
이번 주 일정 보여줘
```

#### 특별 행사 등록

```
특별 행사 등록: 부활절 찬양 예배, 4월 20일 오전 10시, 본당
```

AI가 시설 충돌 여부를 확인하고 행사를 등록합니다.

## 월간 업무

### 업무 1: 월간 재정 보고서

**시기**: 매월 첫째 주
**소요 시간**: 30~45분 (검토 시간 포함)

**중요**: 재정 보고서는 **이중 리뷰**가 필요합니다 — 재정 담당 집사와 담임목사 모두 승인해야 합니다. AI는 재정 관련 산출물을 자동 승인하지 않습니다.

#### 보고서 생성

```
이번 달 재정 보고서 만들어줘
```

AI가 다음을 수행합니다:
1. 항목별 헌금 요약 (십일조, 감사헌금, 특별헌금, 기타)
2. 항목별 지출 목록 (관리비, 인건비, 사역비, 선교비, 기타)
3. 예산 대비 실적 비교
4. 개별 기부금영수증 생성

#### 검토 절차

1. **행정 간사 검토**: 금액을 교회 기록과 대조 확인
2. **재정 담당 집사 검토**: 재정 정확성 확인
3. **담임목사 승인**: 최종 결재

---

### 업무 2: 새신자 단계 검토

**시기**: 매월
**소요 시간**: 15~20분

모든 새신자를 검토하고, 마일스톤을 달성한 분들의 단계를 전환합니다:

```
새신자 단계 현황 보여줘
```

단계 전환이 가능한 각 새신자에 대해:

```
새신자 단계 변경: [이름] → [다음 단계]
```

예시: "새신자 단계 변경: 박지민 → 소그룹 연결"

---

### 업무 3: 교인 정보 갱신

**시기**: 수시
**소요 시간**: 건당 2~5분

교인 검색 김철수

## Inbox 폴더 사용법

`inbox/` 폴더는 데이터를 시스템에 넣는 가장 쉬운 방법입니다. 해당하는 하위 폴더에 파일을 넣기만 하면 됩니다:

### 폴더 구조

```
inbox/
├── documents/    ← Drop Excel/Word files here
├── images/       ← Drop namecard photos, document scans
├── templates/    ← Drop template images for scan-and-replicate
├── staging/      ← Files being processed (don't touch)
├── processed/    ← Successfully processed files
└── errors/       ← Files that couldn't be processed
```

### 파일 넣으면 어떻게 되나요?

| 파일 형식 | 넣을 곳 | 처리 결과 |
|----------|---------|----------|
| Excel (.xlsx, .xls) | documents/ | 1단계 파서가 데이터를 추출 → 검증 → 가져오기 |
| Word (.docx) | documents/ | 1단계 파서가 텍스트를 추출 → 분류 |
| CSV (.csv) | documents/ | 1단계 파서가 데이터를 읽기 → 검증 → 가져오기 |
| 이미지 (.jpg, .png) | images/ | 2단계 파서가 텍스트를 인식 → 기록 생성 |
| PDF (.pdf) | documents/ | 1단계 파서가 내용을 추출 → 분류 |
| 템플릿 이미지 | templates/ | 스캔-앤-복제로 template.yaml 생성 |
| 인식 불가 형식 | → errors/ | 설명과 함께 errors/로 이동 |

### 파일을 넣은 후

Claude를 시작하고 처리를 요청합니다:

```
inbox 처리해줘
```

AI가 다음을 수행합니다:
1. 모든 inbox 폴더 스캔
2. 적절한 단계의 파서로 각 파일 파싱
3. 추출된 데이터 검증
4. 올바른 데이터 파일에 가져오기
5. 처리된 파일을 `inbox/processed/`로 이동
6. 실패한 파일은 오류 설명과 함께 `inbox/errors/`로 이동

## AI 결과물 이해하기

### 승인 단계

시스템은 위험도에 따라 다른 승인 단계를 적용합니다:

| 업무 | 승인 방식 | 이유 |
|------|----------|------|
| 주보 생성 | 단일 검토 (행정 간사) | 낮은 위험 — 서식 작업만 포함 |
| 새신자 등록 | 자동 + 행정 간사 검토 | 중간 위험 — 새 데이터 입력 |
| 정착 단계 전환 | 행정 간사 명시적 승인 | 중간 위험 — 돌봄 관련 결정 |
| 재정 보고서 | 이중 리뷰 (행정 간사 + 재정 담당 집사 + 담임목사) | 높은 위험 — 재정 데이터 |
| 공식 문서 | 단일 검토 (행정 간사) | 중간 위험 — 교회 대외 문서 |

### "PASS"와 "FAIL"의 의미

AI가 검증 결과를 보여줄 때:
- **PASS**: 데이터가 정확하고 일관성이 있음
- **FAIL**: 주의가 필요한 사항이 있음 — AI가 무엇이 문제인지 설명합니다

검증이 실패하면 계속 진행하기 전에 해당 문제를 수정하세요. AI가 안내해 드립니다.

## 효과적으로 사용하는 요령

1. **구체적으로 말하기**: "주보"보다 "이번 주 주보 만들어줘"가 더 좋습니다
2. **세부 사항 포함하기**: "공지 추가"보다 "공지사항 추가: 성가대 연습 시간이 토요일 오후 3시로 변경됩니다"가 더 좋습니다
3. **승인 전 반드시 확인하기**: AI 결과물을 승인하기 전에 항상 검토하세요, 특히 재정 관련은 꼭 확인하세요
4. **모르겠으면 도움 요청하기**: "도와줘"라고 입력하면 AI가 가능한 작업을 설명합니다
5. **문제 보고하기**: 무언가 이상하면 IT 자원봉사자에게 알려주세요
